Class TermOrCurie

source code

Wrapper around vocabulary management, ie, mapping a term to a URI, as well as a CURIE to a URI. Each instance of this class belongs to a "state", instance of state.ExecutionContext. Context definitions are managed at initialization time.

(In fact, this class is, conceptually, part of the overall state at a node, and has been separated here for an easier maintenance.)

The class takes care of the stack-like behavior of vocabulary items, ie, inheriting everything that is possible from the "parent". At initialization time, this works through the prefix definitions (i.e., @prefix or @xmln: attributes) and/or @vocab attributes.

CURIE_to_URI(self, val)

source code 

CURIE to URI mapping.

This method does not take care of the last step of CURIE processing, ie, the fact that if it does not have a CURIE then the value is used a URI. This is done on the caller's side, because this has to be combined with base, for example. The method does take care of BNode processing, though, ie, CURIE-s of the form "_:XXX".

Parameters:
  • val (string) - the full CURIE
Returns:
URIRef of a URI or None.
