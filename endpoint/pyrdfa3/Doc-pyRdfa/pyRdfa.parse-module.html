Module parse

source code

The core parsing function of RDFa. Some details are put into other modules to make it clearer to update/modify (e.g., generation of @property values, or managing the current state).

Note that the entry point (parse_one_node) bifurcates into an RDFa 1.0 and RDFa 1.1 version, ie, to _parse_1_0 and _parse_1_1. Some of the parsing details (management of @property, list facilities, changed behavior on @typeof)) have changed between versions and forcing the two into one function would be counter productive.

parse_one_node(node, graph, parent_object, incoming_state, parent_incomplete_triples)

source code 

The (recursive) step of handling a single node.

This entry just switches between the RDFa 1.0 and RDFa 1.1 versions for parsing. This method is only invoked once, actually, from the top level; the recursion then happens in the _parse_1_0 and _parse_1_1 methods for RDFa 1.0 and RDFa 1.1, respectively.

Parameters:
  • node - the DOM node to handle
  • graph (RDFLib's Graph object instance) - the RDF graph
  • parent_object - the parent's object, as an RDFLib URIRef
  • incoming_state (state.ExecutionContext) - the inherited state (namespaces, lang, etc.)
  • parent_incomplete_triples - list of hanging triples (the missing resource set to None) to be handled (or not) by the current node.
Returns: Boolean
whether the caller has to complete it's parent's incomplete triples
