  2  """ 
  3  Host language sub-package for the pyRdfa package. It contains variables and possible modules necessary to manage various RDFa 
  4  host languages. 
  5   
  6  This module may have to be modified if a new host language is added to the system. In many cases the rdfa_core as a host language is enough, because there is no need for a special processing. However, some host languages may require an initial context, or their value may control some transformations, in which case additional data have to be added to this module. This module header contains all tables and arrays to be adapted, and the module content may contain specific transformation methods.

 14  U{W3C® SOFTWARE NOTICE AND LICENSE<href="http://www.w3.org/Consortium/Legal/2002/copyright-software-20021231">} 
 15   
 16  @var content_to_host_language: a dictionary mapping a media type to a host language 
 17  @var preferred_suffixes: mapping from preferred suffixes for media types; used if the file is local, ie, there is not HTTP return value for the media type. It corresponds to the preferred suffix in the media type registration 
 18  @var initial_contexts: mapping from host languages to list of initial contexts 
 19  @var accept_xml_base: list of host languages that accept the xml:base attribute for base setting 
 20  @var accept_xml_lang: list of host languages that accept the xml:lang attribute for language setting. Note that XHTML and HTML have some special rules, and those are hard coded... 
 21  @var warn_xmlns_usage: list of host languages that should generate a warning for the usage of @xmlns (for RDFa 1.1) 
 22  @var accept_embedded_rdf_xml: list of host languages that might also include RDF data using an embedded RDF/XML (e.g., SVG). That RDF data may be merged with the output 
 23  @var accept_embedded_turtle: list of host languages that might also include RDF data using a C{script} element. That RDF data may be merged with the output 
 24  @var require_embedded_rdf: list of languages that must accept embedded RDF, ie, the corresponding option is irrelevant 
 25  @var host_dom_transforms: dictionary mapping a host language to an array of methods that are invoked at the beginning of the parsing process for a specific node. That function can do a last minute change on that DOM node, eg, adding or modifying an attribute. The method's signature is (node, state), where node is the DOM node, and state is the L{Execution context<pyRdfa.state.ExecutionContext>}. 
 26  @var predefined_1_0_rel: terms that are hardcoded for HTML+RDF1.0 and replace the initial context for that version 
 27  @var beautifying_prefixes: this is really just to make the output more attractive: for each media type a dictionary of prefix-URI pairs that can be used to make the terms look better...
