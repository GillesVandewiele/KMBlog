Class ExecutionContext

source code

State at a specific node, including the current set of namespaces in the RDFLib sense, current language, the base, vocabularies, etc. The class is also used to interpret URI-s and CURIE-s to produce URI references for RDFLib.

__init__(self, node, graph, inherited_state=None, base='', options=None, rdfa_version=None)
(Constructor)

source code 
Parameters:
  • node - the current DOM Node
  • graph - the RDFLib Graph
  • inherited_state (state.ExecutionContext) - the state as inherited from upper layers. This inherited_state is mixed with the state information retrieved from the current node.
  • base - string denoting the base URI for the specific node. This overrides the possible base inherited from the upper layers. The current XHTML+RDFa syntax does not allow the usage of @xml:base, but SVG1.2 does, so this is necessary for SVG (and other possible XML dialects that accept @xml:base)
  • options (Options) - invocation options, and references to warning graphs

_URI(self, val)

source code 

Returns a URI for a 'pure' URI (ie, not a CURIE). The method resolves possible relative URI-s. It also checks whether the URI uses an unusual URI scheme (and issues a warning); this may be the result of an uninterpreted CURIE...

add_to_list_mapping(self, property, resource)

source code 

Add a new property-resource on the list mapping structure. The latter is a dictionary of arrays; if the array does not exist yet, it will be created on the fly.

Parameters:
  • property - the property URI, used as a key in the dictionary
  • resource - the resource to be added to the relevant array in the dictionary. Can be None; this is a dummy placeholder for <span rel="property" inlist>...</span> constructions that may be filled in by children or siblings; if not an empty list has to be generated.
